# REV-RUNTIME-IDENTITY-BOOTSTRAP-2026-06-21

Statut: closed

## Scope

Re-review de la vague qui supprime l'allocateur manuel `LOCK TABLE` + `MAX(id) + 1`
dans `astral_calculator/src/infra/db/runtime_queries/calculation.rs` et de son
bootstrap PostgreSQL associe.

## Findings initiaux

- High: les ecritures runtime calculaient encore les `id` via scan de table,
  ce qui serialisait les writers et rendait la persistance fragile sous
  concurrence.
- Medium: le bootstrap PostgreSQL n'avait pas de garde automatisee garantissant
  que les tables runtime restent a identite native.

## Corrections

- `calculation.rs` insere maintenant les lignes runtime sans colonne `id` et
  recupere l'identifiant cree par PostgreSQL avec `RETURNING id` pour
  `astral_chart_calculations`.
- `scripts/import_json_db_to_postgres.py` marque les six tables runtime a
  `id` auto-genere via `generated by default as identity`.
- `tests/runtime_identity_bootstrap_tests.rs` interdit a la fois le retour de
  `next_id` / `LOCK TABLE` / `MAX(id)` dans les queries runtime et la perte de
  la liste `IDENTITY_TABLES` dans le bootstrap.

## Re-review

Aucun finding ouvert.
